SyntaxHighlighter 1.1.2

About

SyntaxHighlighter is a Formatter that displays code attractively with syntax highlighting in pre-formatted (monospace) blocks. Each code block is accompanied by "view plain", "copy to clipboard", and "print" links.

Requirements

I've only tested this on Vanilla 1.1.4, but I believe that it will work with any version of Vanilla 1. I'll appreciate any user feedback regarding compatibility. This extension doesn't depend on any other extensions.

Installation

Unpack the SyntaxHighlighter folder and move it to your forum's extensions directory. The default.php file should be at:

/extensions/SyntaxHighlighter/default.php

Then, while logged into your forum as an administrator, go to Settings > Extensions, check off the box to the left of "Syntax Highlighter" in the Extensions table.

Using SyntaxHighlighter

When you want to highlight syntax in a post, select SyntaxHighlighter for Format comments as while posting a new discussion, reply, or other message.

In the message or post itself, wrap any code in [code] blocks. For example, to highlight some CSS code:

[code lang="css"]
p {
    padding: 0;
    font-family: arial, sans-serif;
    font-size: 1.1em;
    line-height: 1.5;
    margin: 1em 0;
}
[/code]

The following languages can be highlighted with their corresponding "lang" values:

Language "lang" Value
C++ cpp, c, c++
C# c#, c-sharp, csharp
CSS css
Delphi delphi, pascal
Java java
JavaScript js, jscript, javascript
PHP php
Python py, python
Ruby rb, ruby, rails, ror
Sql sql
VB vb, vb.net
XML/HTML xml, html, xhtml, xslt

Other languages which aren't bundled with this extension can be added. To do so will require editing the extension files. See the SyntaxHighlighter Wiki for direction.

To load fewer languages (for faster page loads), comment out any languages in the $hlScripts array in this extension's default.php. For example, to disable Delphi support:

/**
 * Load the language scripts. Comment out any languages you don't need.
 */
$hlScripts = array(
    'shCore.js', // REQUIRED
    'shBrushCpp.js', // C++
    'shBrushCSharp.js', // C# (C-Sharp)
    'shBrushCss.js', // CSS
 // 'shBrushDelphi.js', // Delphi
    'sBrushJava.js', // Java
    'shBrushJScript.js', // JavaScript
    'shBrushPhp.js', // PHP
    'shBrushPython.js', // Python
    'shBrushRuby.js', // Ruby
    'shBrushSql.js', // SQL
    'shBrushVb.js', // VB (Visual Basic)
	'shBrushXml.js', // XML
);

Note the slashes, //, which comment out a line. If you comment the second line, 'shCore.js', SyntaxHighlighter won't work, so use care when making modifications like this.

Extending SyntaxHighlighter

I don't recommend extending SyntaxHighlighter. It was written with a narrow goal, and as such doesn't have any facilities that StringFormatter lacks, other than the Parse() method that replaces [code] blocks.
